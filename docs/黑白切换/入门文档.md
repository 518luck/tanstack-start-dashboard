# 整体布局

1. 黑白模式切换：light / dark
2. 皮肤主题切换：vercel / mono / claude / ...

# 一、黑白主题怎么切

- 根入口在 src/routes/\_\_root.tsx:85。这里包了 next-themes 的 ThemeProvider，配置是 attribute="class"，
- 所以切换黑白模式时，next-themes 会给根节点加上 .dark class。对应按钮在 src/components/themes/theme- mode-toggle.tsx:9，核心就是 setTheme('light' | 'dark')。它还配了 View Transition 动画，点击时先写入 --x / --y，再调用 document.startViewTransition(...)；动画样式在 src/styles/lobals.css:70。

## 这层的样式消费方式主要有两种：

- 用语义色类，比如 bg-background、text-foreground
- 用 dark: 写暗色差异，比如 src/components/ui/button.tsx:13 里的 dark:bg-input/30、dark:hover:bg- input/50

# 二、页面里的“各种主题”怎么切

这是项目自定义的一层，不是 next-themes。入口在 src/components/themes/active-theme.tsx:20。它维护
activeTheme，然后把当前主题写到 html 的 data-theme 属性上：

- 服务端首屏先从 cookie 读 active_theme，见 src/routes/\_\_root.tsx:24
- 客户端切换后再把新值写回 cookie，见 src/components/themes/active-theme.tsx:30
- 选择器组件在 src/components/themes/theme-selector.tsx:17
- 可选主题列表在 src/components/themes/theme.config.ts:5

所以这层不是切 .dark，而是切 html[data-theme="vercel"]、html[data-theme="mono"] 这种属性。

三、页面为什么会跟着变
关键在主题 CSS。总入口是 src/styles/theme.css:1，它把所有主题文件都引进来。比如 src/styles/themes/
vercel.css:1：

- [data-theme='vercel'] 定义浅色变量
- [data-theme='vercel'].dark 定义这个皮肤下的深色变量
- 后面的 @theme inline 再把这些变量映射成 Tailwind 语义 token，如 --color-background、--color-
  card、--color-primary

所以最终页面组件几乎不需要知道“当前是什么主题”，只写语义类就行，比如：

- src/features/overview/components/overview.tsx:37 用了 from-primary/5、to-card
- src/components/layout/header.tsx:13 用了 bg-background/60
- src/components/kbar/index.tsx:66 用了 bg-background/80、bg-card

也就是说，页面内“各种主题切换”的本质不是页面自己切，而是所有页面都统一消费全局 token；一旦 data-
theme 或 .dark 变了，整个页面自然一起变。

你可以按这个顺序学

1. 先看 src/routes/\_\_root.tsx:61，理解 .dark 和 data-theme 两层入口。
2. 再看 src/components/themes/active-theme.tsx:20，理解自定义主题状态和 cookie 持久化。
3. 再挑一个主题文件，比如 src/styles/themes/vercel.css:1，看变量如何定义。
4. 最后看业务组件如何只写 bg-background、text-foreground、dark:，比如 src/components/ui/
   button.tsx:8。
