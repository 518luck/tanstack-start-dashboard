# 从 0 搭建

## 一、先理解主题系统的本质

不要一开始就想着“切换按钮”，先想清楚这几个层：

1. 主题定义层  
   你要先规定有哪些主题，比如 light、dark、vercel、claude。
2. 主题状态层  
   当前用户选的是哪个主题，要放在 React state / context / store 里。
3. 主题落地层  
   把“当前主题名”同步到 DOM，比如：

   ```html
   <html data-theme="vercel"></html>
   ```

4. 样式映射层  
   不同主题下，用 CSS 变量定义不同颜色、字体、圆角：

   ```css
   [data-theme="vercel"] {
     --background: white;
     --foreground: black;
   }

   [data-theme="claude"] {
     --background: #f5f1e8;
     --foreground: #2b241d;
   }
   ```

5. 持久化层  
   用户刷新页面后，主题不能丢，所以要存 localStorage 或 cookie。

这 5 层齐了，主题系统才算完整。

## 二、从 0 搭建时的推荐做法

最稳的方案是：data-theme + CSS variables + Context

### Step 1：先定义主题列表

先统一管理主题名，不要把字符串写得到处都是。

```ts
export const THEMES = [
  { name: "Vercel", value: "vercel" },
  { name: "Claude", value: "claude" },
  { name: "Mono", value: "mono" },
];

export const DEFAULT_THEME = "vercel";
```

重点：

- name 给界面显示
- value 给程序判断
- DEFAULT_THEME 给首次进入的用户

### Step 2：用 CSS 变量做“主题令牌”

不要在组件里写死颜色，比如不要这样：

```tsx
className = "bg-white text-black";
```

而是写成：

```tsx
className = "bg-background text-foreground";
```

然后让 background、foreground 对应 CSS 变量。

例如：

```css
[data-theme="vercel"] {
  --background: white;
  --foreground: black;
  --primary: black;
}

[data-theme="claude"] {
  --background: #f7f3eb;
  --foreground: #3a3128;
  --primary: #c27d3b;
}
```

这是最关键的技巧：  
组件永远只消费“语义变量”，不要直接消费具体颜色。

### Step 3：把主题挂到 html 上

切换主题时，不是给每个组件单独改 class，而是统一改根节点：

```ts
document.documentElement.setAttribute("data-theme", theme);
```

这样整个页面的 CSS 变量都会跟着变。

这是主题系统比“条件渲染 className”更适合大项目的原因。

### Step 4：做一个 ThemeProvider

你需要一个全局 provider 管主题状态。

思路类似：

```tsx
const ThemeContext = createContext(...)

function ThemeProvider({ children }) {
  const [theme, setTheme] = useState(DEFAULT_THEME)

  useEffect(() => {
    document.documentElement.setAttribute("data-theme", theme)
    localStorage.setItem("theme", theme)
  }, [theme])

  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      {children}
    </ThemeContext.Provider>
  )
}
```

这样任何组件都能：

```tsx
const { theme, setTheme } = useTheme();
```

### Step 5：初始化时恢复用户主题

如果只在 useEffect 里恢复，页面第一次渲染可能会闪一下默认主题，这叫 FOUC。

更好的做法有两个：

1. CSR 项目  
   页面一加载就先读 localStorage，再把 data-theme 提前写到 html
2. SSR 项目  
   服务端从 cookie 读取主题，首屏 HTML 直接输出：

   ```html
   <html data-theme="vercel"></html>
   ```

你这个项目就是偏第二种：服务端先读 cookie，再把 data-theme={activeTheme} 放到根节点上，所以首屏不容易闪。

## 三、实际开发里的文件组织建议

从 0 搭时，建议这样拆：

- theme.config.ts  
  管主题列表、默认主题
- theme-provider.tsx  
  管 context、状态、切换逻辑、持久化
- theme.css  
  管主题基础入口
- themes/vercel.css
- themes/claude.css
- themes/mono.css

也就是：

- JS/TS 负责“当前主题是谁”
- CSS 负责“这个主题长什么样”

这个职责分离很重要。

## 四、你真正要掌握的不是换色，而是“设计令牌”

成熟的主题系统，不只换颜色，还会换：

- 背景色
- 文字色
- 边框色
- 主按钮色
- 阴影
- 圆角
- 字体
- 间距密度
- 图表颜色
- Sidebar 专属颜色

比如你这个项目里，每个主题文件不只是 --background，还有：

- --primary
- --muted
- --sidebar
- --chart-1
- --font-sans
- --radius
- --shadow-\*

这就是从“换皮肤”升级到“设计系统主题化”的关键。

你从 0 学的时候，建议先定义最小集合：

```txt
--background
--foreground
--primary
--secondary
--border
--muted
--radius
```

等这套跑通，再慢慢扩。

## 五、最常见的 4 个坑

1. 组件里写死颜色  
   这是最大的问题。只要你写死 bg-white、text-black，主题系统就会失效一部分。
2. 主题名到处散落  
   不要在多个文件里重复写 "vercel"、"dark"，统一放配置。
3. 首屏闪烁  
   如果主题是渲染后才恢复，用户会看到先亮后暗，或者先默认再切换。
4. 混淆“颜色模式”和“品牌主题”  
   这是高级一点但很重要的点。

其实你可以把主题拆成两维：

- 模式：light / dark
- 风格：vercel / claude / mono

这样组合后会更强，比如：

- vercel + dark
- claude + light

你这个项目其实已经有这个思路了，因为每个主题文件里还有：

```css
[data-theme='vercel'].dark { ... }
```

说明它支持“同一主题下的暗色版本”。

## 六、推荐你的学习路径

如果你要自己从 0 实现，我建议你按这个顺序练：

1. 先做 2 个主题，只切换背景和文字
2. 再引入 ThemeProvider
3. 再做持久化 localStorage
4. 再解决首屏闪烁
5. 再把颜色扩成完整 token
6. 最后再做主题切换 UI，比如 dropdown、command、快捷键

也就是说：  
先把“系统”搭起来，再做“界面”。

## 七、一个最小可行版本的思路

你可以先做成这样：

```ts
// 切换时
setTheme("claude");
document.documentElement.setAttribute("data-theme", "claude");
localStorage.setItem("theme", "claude");
```

```css
[data-theme="vercel"] {
  --background: white;
  --foreground: black;
}

[data-theme="claude"] {
  --background: #f7f3eb;
  --foreground: #2f2923;
}

body {
  background: var(--background);
  color: var(--foreground);
}
```

这就是最小闭环。
